Convert OptimalControlProblem on \( (\mathbb{G}, \mathbb{U}) \) into a tangent space QuadraticProgramSparse on \( (\mathbb{R}^{\dim \mathfrak{g}}, \mathbb{R}^{\dim \mathfrak{u}}) \).

The OptimalControlProblem is encoded into a QuadraticProgram via linearization around \((g_{lin}(t), u_{lin}(t))\) followed by time discretization. The variables of the QP are

\[ \begin{bmatrix} \mu_0 & \mu_1 & \ldots & \mu_{K - 1} & x_1 & x_2 & \ldots & x_K \end{bmatrix}, \]

where the discrete time index \(k\) corresponds to time \(t_k = k \frac{T}{K} \) for \( k = 0, 1, \ldots, K \).

performance.

Template Parameters
Gproblem state group type \( \mathbb{G} \)
Uproblem input group type \( \mathbb{U} \)
Dyndynamics functor type
DTdifferentiation method to utilize
Parameters
pbmoptimal control problem
Knumber of discretization points. More points create a larger QP, but the distance \( T / K \) between points should be smaller than the smallest system time constant for adequate
fdynamics \( f : \mathbb{R} \times \mathbb{G} \times \mathbb{U} \rightarrow \mathbb{R}^{\dim \mathfrak g}\) s.t. \( \mathrm{d}^r g_t = f(t, g, u) \)
linlinearization point
Returns
QuadraticProgramSparse modeling the input optimal control problem.
Note
Given a solution \((x^*, \mu^*)\) to the QuadraticProgramSparse, the corresponding solution to the OptimalControlProblem is \( u^*(t) = u_{lin}(t) \oplus \mu^*(t) \) and the optimal trajectory is \( g^*(t) = g_{lin}(t) \oplus x^*(t) \).
Constraints are added as \( A x \leq b - A (g_{lin} - c) \) and similarly for the input. Beware of using constraints on non-Euclidean spaces.
f must be differentiable w.r.t. \( g \) and \( u \) with the default smooth::diff method (check smooth::diff::DefaultType). If using an automatic differentiation method this means that it must be templated on the scalar type.
