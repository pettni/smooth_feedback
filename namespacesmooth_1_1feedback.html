const ASIFtoQPParams &  prm,
Dyn &&  f,
SS &&  h,
BackupU &&  bu 
)

Convert an ASIFProblem to a QuadraticProgram.

The objective is to impose constraints on the current input \( u \) of a system \( \mathrm{d}^r x_t = f(x, u) \) s.t.

\[ \frac{\mathrm{d}}{\mathrm{d}t} h(\phi(t; x_0, bu(\cdot))) \geq \alpha h(\phi(t; x_0, bu(\cdot))) \]

which enforces forward invariance of the set \( \{ x : h(t, x) \geq 0 \} \) along the backup trajectory \( bu \). The constraint is enforced at \(K\) look-ahead time steps \( t_k = k \tau\) for \(k = 0, \ldots, K \).

This function encodes the problem as a QuadraticProgram that solves

\[ \begin{cases} \min_{\mu} & \left\| u - u_{des} \right\|^2 \\ \text{s.t.} & \text{constraint above holds for } u = u_{des} + \mu \end{cases} \]

A solution \( \mu^* \) to the QuadraticProgram corresponds to an input \( u_{des} \oplus \mu^* \) applied to the system.

Template Parameters
Gstate LieGroup type \(\mathbb{G}\)
Uinput Manifold type \(\mathbb{G}\)
Parameters
pbmproblem definition
prmalgorithm parameters
fsystem model \(f : \mathbb{R} \times \mathbb{G} \times \mathbb{U} \rightarrow \mathbb{R}^{\dim \mathfrak g}\) s.t. \( \mathrm{d}^r x_t = f(t, x, u) \)
hsafe set \(h : \mathbb{R} \times \mathbb{G} \rightarrow \mathbb{R}^{n_h}\) s.t. \( S(t) = \{ h(t, x) \geq 0 \} \) denotes the safe set at time \( t \)
bubackup controller \(ub : \mathbb{R} \times \mathbb{G} \rightarrow \mathbb{U} \)
Note
The algorithm relies on automatic differentiation. The following supplied functions must be differentiable (i.e. be templated on the scalar type if an automatic differentiation method is selected):
  • f differentiable w.r.t. x and u
  • h differentiable w.r.t. t and x
  • bu differentiable w.r.t. x
Returns
QuadraticProgram modeling the ASIF filtering problem

template<LieGroup G, Manifold U, typename Dyn , diff::Type DT = diff::Type::DEFAULT>
QuadraticProgramSparse<double> smooth::feedback::ocp_to_qp ( const OptimalControlProblem< G, U > &  pbm,
std::size_t  K,
const Dyn &  f,
const LinearizationInfo< G, U > &  lin 
)

Convert OptimalControlProblem on \( (\mathbb{G}, \mathbb{U}) \) into a tangent space QuadraticProgramSparse on \( (\mathbb{R}^{\dim \mathfrak{g}}, \mathbb{R}^{\dim \mathfrak{u}}) \).

The OptimalControlProblem is encoded into a QuadraticProgram via linearization around \((g_{lin}(t), u_{lin}(t))\) followed by time discretization. The variables of the QP are

\[ \begin{bmatrix} \mu_0 & \mu_1 & \ldots & \mu_{K - 1} & x_1 & x_2 & \ldots & x_K \end{bmatrix}, \]

where the discrete time index \(k\) corresponds to time \(t_k = k \frac{T}{K} \) for \( k = 0, 1, \ldots, K \).

performance.

Template Parameters
Gproblem state group type \( \mathbb{G} \)
Uproblem input group type \( \mathbb{U} \)
Dyndynamics functor type
DTdifferentiation method to utilize
Parameters
pbmoptimal control problem
Knumber of discretization points. More points create a larger QP, but the distance \( T / K \) between points should be smaller than the smallest system time constant for adequate
fdynamics \( f : \mathbb{R} \times \mathbb{G} \times \mathbb{U} \rightarrow \mathbb{R}^{\dim \mathfrak g}\) s.t. \( \mathrm{d}^r g_t = f(t, g, u) \)
linlinearization point
Returns
QuadraticProgramSparse modeling the input optimal control problem.
Note
Given a solution \((x^*, \mu^*)\) to the QuadraticProgramSparse, the corresponding solution to the OptimalControlProblem is \( u^*(t) = u_{lin}(t) \oplus \mu^*(t) \) and the optimal trajectory is \( g^*(t) = g_{lin}(t) \oplus x^*(t) \).
Constraints are added as \( A x \leq b - A (g_{lin} - c) \) and similarly for the input. Beware of using constraints on non-Euclidean spaces.
f must be differentiable w.r.t. \( g \) and \( u \) with the default smooth::diff method (check smooth::diff::DefaultType). If using an automatic differentiation method this means that it must be templated on the scalar type.
